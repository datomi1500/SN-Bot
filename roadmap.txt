Resumen de la estructura

Objetivo
- Mantener esto como un mapa práctico del código fuente.
- Mostrar únicamente las carpetas principales y los archivos importantes (no todos los archivos).

Raíz
- index.js                    -> punto de entrada del proceso
- package.json                -> scripts, dependencias y metadatos del proyecto
- README.md                   -> documentación de configuración y uso
- SECURITY.md                 -> política de seguridad
- scripts/                    -> scripts de utilidades y mantenimiento (copias de seguridad, restauración, migración)
- logs/                       -> registros de ejecución
- src/                        -> código principal de la aplicación del bot
- tests/                      -> pruebas unitarias (se ejecutan mediante «npm test»)

src/
- app.js                      -> crea el cliente, inicializa los sistemas e inicia el servidor web

- commands/                   -> categorías de comandos de barra inclinada por función
  - Core/                     -> ayuda, ping, estadísticas, tiempo de actividad, invitación
  - Community/                -> gestión de aplicaciones + paneles de control
  - Logging/                  -> registros de auditoría + gestión de eventos
    - modules/logging_dashboard.js
    - modules/logging_setchannel.js
  - Ticket/                   -> sistema de tickets de soporte + panel de control
  - ServerStats/              -> sistema de canales de estadísticas del servidor
  - Economy/                  -> sistema económico
  - Moderation/               -> comandos de moderación y administración
  - Verification/             -> flujo de verificación de miembros
  - JoinToCreate/             -> creación dinámica de canales de voz
  - Reaction_roles/           -> asignación automática de roles
  - Cumpleaños/, Diversion/, Sorteo/, Niveles/, Búsqueda/, Herramientas/, Utilidades/, Bienvenida/

- config/                     -> módulos de configuración centralizados
- events/                     -> Detectores de eventos de Discord
- handlers/                   -> Enrutamiento de interacciones y controladores de componentes
  - commandLoader.js          -> Carga los comandos de barra al iniciarse
  - events.js                 -> Registro de detectores de eventos de Discord
  - interactions.js           -> Enrutador de botones, menús desplegables y ventanas modales
  - ticketButtons.js, reactionRoles.js, etc.

- interactions/               -> controladores de botones, modales y menús de selección
- services/                   -> lógica de negocio + operaciones con datos
- utils/                      -> ayudantes compartidos
  - database.js               -> fachada que reexporta los módulos de la base de datos
  - database/wrapper.js       -> DatabaseWrapper, ayudantes get/set
  - database/keys.js          -> Generadores de claves al estilo Redis
  - database/tickets.js       -> Persistencia de tickets + estadísticas de tickets de gremios
  - panelStatus.js            -> Detección compartida de mensajes del panel
  - errorHandler.js, logger.js, embeds.js, etc.

Base de datos
- PostgreSQL (principal) con alternativa en memoria para el desarrollo
- Degradación gradual cuando PostgreSQL no está disponible

Notas
- Modular por área funcional: comandos -> controladores -> servicios -> utilidades
- bot.js es la única fuente de referencia para la personalización de temas y colores de los elementos incrustados y la configuración básica del bot
